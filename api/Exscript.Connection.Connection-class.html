Class Connection

object --+
         |
        Connection

This class is a decorator for protocols.Transport objects that adds thread safety by adding locking to the authenticate() and authorize() functions. It also provides access to the associated Queue and Host instances.

For complete documentation, please refer to the protocols.Transport documentation.

app_authenticate(self, account=None, flush=True, lock=True)

 

Like protocols.Transport.app_authenticate(), but makes sure to lock/release the account while it is used. If an account is not given, one is acquired from the account manager. Returns the account that was used to log in.

auto_app_authorize(self, account=None, flush=True, lock=True)

 

Executes a command on the remote host that causes an authorization procedure to be started, then authorizes using app_authorize(). For the meaning of the account, flush, and lock arguments see app_authorize(). If the password argument is given, the given password is used instead of the given account.

Depending on the detected operating system of the remote host the following commands are started:

  • on IOS, the "enable" command is executed.
  • nothing on other operating systems yet.

Returns the account that was used to log in.

Parameters:
  • account (Account) - The account to use for logging in.
  • flush (bool) - Whether to flush the last prompt from the buffer.
  • lock (bool) - Whether to lock the account while logging in.
Returns: Account
The account that was used to log in.
